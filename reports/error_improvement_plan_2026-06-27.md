# 误差改进方案

## 当前问题

当前 10% 点位验证中，随机森林模型相对 IceBridge/IMB 点位真值的 RMSE 约 0.05 m，而 AMSR/SMOS 与 ECCO 外部产品约 0.17 m。这个结果不能简单解释为“外部产品很差”，因为外部产品是格网尺度，点位观测是点尺度，两者存在尺度不一致。

因此后续改进分为两条线：

1. 降低本文模型相对点位真值的误差。
2. 让外部产品对比更加公平、可解释。

## 一、模型误差改进

### 1. 增加质量控制分组

建议分别报告以下子集误差：

- IceBridge 与 IMB 分开评价。
- 冬季/春季冻结期单独评价，弱化融化期浮标异常影响。
- 雪深分段评价：0-0.1 m、0.1-0.3 m、0.3 m 以上。
- 外部产品共同有效点单独评价。

目的：找出误差主要来自哪类样本，而不是只看一个总 RMSE。

### 2. 降低 IMB 连续时间样本的相关性影响

IMB 浮标每天连续漂移，很多样本相邻时间和空间高度相关。随机 90%/10% 划分会让训练集和测试集过于相似。建议增加：

- 按浮标编号留一验证。
- 按年份留一验证。
- 按日期块划分验证。

这类验证的误差可能会上升，但可信度更高。

### 3. 进行源平衡训练

当前 IMB 样本数量较多，模型可能更偏向 IMB 分布。建议训练时加入样本权重：

- IceBridge 和 IMB 总权重相同。
- 不同雪深区间权重相近。

这样可以改善 IceBridge 上的误差，并减少模型只适应某一类点位。

### 4. 尝试两套特征方案

主方案保持“亮温 + 被动微波派生特征”，保证物理解释清晰。

增强方案加入可用于面状反演的辅助变量：

- 纬度、经度
- 年内日序 DOY
- 海冰密集度或冰厚辅助场，如果区域反演时也能获得

注意：加入经纬度和日期可能降低点位误差，但会引入地理/季节先验。汇报时应作为增强实验，不替代主方案。

### 5. 尝试更强模型作为对照

随机森林作为主模型可以保留，但建议增加模型对照：

- ExtraTreesRegressor
- GradientBoosting / HistGradientBoosting
- XGBoost 或 LightGBM，如果服务器环境允许

目标不是盲目追低误差，而是证明随机森林不是偶然结果。

## 二、外部产品对比改进

这一步是当前优先级最高的工作。目标不是强行让外部产品误差变小，而是让比较方式变得公平、可解释，从而可以谨慎地说明“本文模型在当前点位验证数据上优于外部产品”。

### 1. 点位对比改成“同尺度对比”

当前是：

点位真值 vs 最近邻格网产品

更公平的方式是：

同一天、同一产品格网内的多个点位先求平均，再与该格网产品比较。

这样可以减少点-面尺度差异，外部产品误差会更有解释力。

已新增脚本：

```powershell
python random_forest/make_product_comparison_report.py --point-csv reports/combined_regional_framework/holdout_10pct_point_comparison.csv --out-dir reports/product_comparison_diagnostics
```

该脚本会输出三类比较：

- `common_valid_points`：模型和产品在共同有效点上比较。
- `distance_le_20km` / `ecco_sic_ge_0p8`：只保留匹配质量更好的点。
- `grid_aggregated_*deg`：把同一天、同一近似格网内的点位先求平均，再和产品比较，减少点-面尺度差异。

### 2. 加入匹配质量筛选

建议至少报告三组：

- 全部产品有效点。
- 最近邻距离小于 20 km 的点。
- ECCO 海冰密集度大于 0.8 的点。

如果筛选后外部产品误差下降，可以说明原始误差部分来自空间匹配和海冰条件。

### 3. 分开解释 AMSR/SMOS 与 ECCO

AMSR/SMOS：

- 有效点少。
- 产品掩膜严格。
- 可作为卫星产品对比，但样本覆盖有限。

ECCO：

- 覆盖率高。
- 但它是再分析/模型产品，不是 SMOS 卫星产品。
- 适合做高覆盖辅助参考，不适合作为唯一精度标准。

## 三、推荐汇报结论

建议不要写：

“本文模型优于 SMOS/ECCO 产品，因此模型一定更好。”

建议写：

“在 IceBridge/IMB 独立点位验证集上，本文随机森林模型相对点位实测雪深的误差低于外部格网产品。外部产品误差较大可能与点-面尺度差异、产品掩膜、匹配距离和产品变量定义有关。后续将通过同尺度格网聚合、匹配质量筛选和分组验证进一步评估模型稳定性。”

## 四、下一步最值得做的实验

优先级从高到低：

1. 重新跑新版脚本，输出距离和海冰密集度筛选后的产品对比指标。
2. 做产品格网尺度聚合对比，而不是点对格网最近邻对比。
3. 增加源平衡训练，观察 IceBridge 误差是否下降。
4. 增加按年份或按浮标留一验证，检查模型泛化能力。
5. 尝试 ExtraTrees/XGBoost 作为模型对照。
