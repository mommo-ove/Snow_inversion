# 组会汇报草稿：海冰积雪厚度反演初步实验

## 这两周做了什么

上次组会我主要讲了一个随机森林的初步结果，当时是随机划分训练集和测试集，结果看起来比较好。后来我觉得这个结果可能有点偏乐观，因为浮标数据是时间序列，同一个浮标相邻时刻的样本会很像。如果随机切分，训练集和测试集里可能同时包含同一个浮标的相近样本，模型就容易学到浮标内部的规律，而不一定是真的具备跨浮标的泛化能力。

所以这两周主要做了三件事：

1. 整理了随机森林实验脚本，把数据检查、模型训练、结果输出分开。
2. 增加了按浮标留出的验证方式，也就是测试集里的浮标在训练时完全不出现。
3. 对比了只使用亮温特征，以及加入时间、位置、气象特征后的结果。

## 数据情况

目前使用的是 IMB 浮标数据，CSV 一共有 12639 条样本、23 个字段。每一行是一条观测记录，目标变量是：

```text
Snow_Depth_m
```

主要输入特征包括 AMSR 亮温通道：

```text
TB_18V, TB_18H, TB_23V, TB_23H,
TB_36V, TB_36H, TB_89V, TB_89H
```

另外也尝试加入了时间、空间和气象信息：

```text
Year, Month, Day, Hour,
Latitude, Longitude, Distance_Km,
Air_Temp_C, Air_Pressure_mbar
```

本地还有一个 `Master_Dataset_AllFeatures.mat` 文件，是 MATLAB v7.3 格式，不能用 `scipy.io.loadmat` 直接读取，后面改用 `h5py` 查看。里面目前看到一个矩阵：

```text
Final_Data: shape=(19, 8671)
```

## 实验设置

目前做了两个验证方式。

第一个是随机划分：

```text
random_split
```

也就是普通的 80/20 切分。这个可以看模型是否能拟合整体数据，但对时间序列浮标数据来说，结果可能偏乐观。

第二个是按浮标留出：

```text
buoy_held_out
```

也就是测试集里的浮标在训练集中完全不出现。这个更严格，更接近“模型能不能推广到没见过的浮标”。

## 结果

只使用亮温和衍生亮温特征时：

```text
random_split:
RMSE = 0.0568 m
MAE  = 0.0191 m
R2   = 0.879

buoy_held_out:
RMSE = 0.2116 m
MAE  = 0.1756 m
R2   = -4.33
held-out buoys = 2014B, 2015E, 2015I
```

加入时间、位置和气象特征后：

```text
random_split:
RMSE = 0.0140 m
MAE  = 0.0079 m
R2   = 0.994

buoy_held_out:
RMSE = 0.1598 m
MAE  = 0.1393 m
R2   = -3.00
held-out buoys = 2014B, 2015A, 2015E
```

## 怎么理解

随机划分下结果很好，说明随机森林确实能从亮温以及环境信息里拟合雪深变化。特别是加入时间、位置和气象信息后，随机划分的 R2 接近 0.99。

但是按浮标留出之后，结果明显下降，R2 甚至变成负数。这说明模型对没见过的浮标泛化能力还不够。换句话说，随机划分的高分不一定代表模型真的学到了稳定的物理反演关系，可能有一部分是在记忆特定浮标、特定时间段或特定区域的统计特征。

我觉得这个结果不是坏事，反而说明后面有明确的改进方向：不能只看随机切分结果，需要更关注跨浮标、跨区域或者跨数据源验证。

## 目前结论

目前可以得到一个比较稳的阶段性结论：

```text
随机森林可以作为海冰积雪厚度反演的 baseline。
在随机划分下模型效果较好，但在按浮标留出的验证中泛化较差。
加入时间、空间和气象特征可以降低误差，但还没有完全解决跨浮标分布差异问题。
```

这个结论也说明后续引入更强的特征工程、跨数据源训练，或者 PINN/物理约束模型是有必要的。

## 下一步计划

下一步我准备做三件事：

1. 做 leave-one-buoy-out 实验，每次只留出一个浮标，看看具体是哪些浮标最难预测。
2. 继续解析 `Master_Dataset_AllFeatures.mat`，确认 19 个变量的物理含义。
3. 尝试做跨数据源验证，比如用一个数据源训练，用另一个数据源验证，看随机森林在冰桥/浮标之间能不能迁移。

如果这些 baseline 的跨源效果仍然不好，再考虑参考老师给的 PINN 代码，把物理约束或先验关系加入模型。

## 命令小抄

查看当前 Git 状态：

```powershell
git status
```

它只是查看，不会改文件。主要看有没有文件被修改、有没有 commit 没 push。

保存一版本地修改：

```powershell
git add .
git commit -m "message"
```

`git add` 是把文件放进暂存区，`git commit` 是把这次改动正式保存成一个版本。

上传到 GitHub：

```powershell
git push
```

这个会把本地 commit 上传到 GitHub。

从 GitHub 拉取最新代码：

```powershell
git pull
```

这个会把 GitHub 上的新 commit 拉到当前电脑或服务器。

服务器访问 GitHub 前，如果网络不稳，需要先在本机开反向隧道：

```powershell
ssh -N -R 127.0.0.1:17897:127.0.0.1:7897 203
```

可以理解为：

```text
远程服务器 127.0.0.1:17897
通过 SSH 隧道
连到本机 127.0.0.1:7897 代理
再访问 GitHub
```

运行数据检查：

```powershell
python random_forest\inspect_data.py
```

运行随机森林实验：

```powershell
python random_forest\run_rf_experiments.py
```

运行加入时间、位置、气象特征的实验：

```powershell
python random_forest\run_rf_experiments.py --include-context --out-dir random_forest\outputs_context
```
