# 组会笔记 2026-04-26

## 这次我想讲什么

这次不打算再把重点放在“随机森林又跑了一次而且分数很高”这种事情上，因为那个故事已经讲过了，而且随机切分的高分现在看起来不太可靠。

这次我想讲的核心是：

```text
我把验证方式做严格了，结果发现模型对随机切分和跨浮标验证非常敏感。
所以现在更关键的问题不是继续刷随机切分指标，而是看模型到底能不能泛化。
```

## 这两周做了什么

这两周我主要做了三件事：

1. 把随机森林实验流程整理了一下，脚本拆成了数据检查、训练和结果输出几部分。
2. 比较了三组输入特征：
   - 只用亮温和衍生亮温特征
   - 加入时间、位置和气象信息
   - 加入时间、位置和气象，但去掉气压
3. 新做了 leave-one-buoy-out，也就是一次留一个浮标做测试，专门看不同浮标的泛化差异。

## 这次讲的实验设置

### 1. tb only

只用亮温和衍生亮温特征：

```text
TB_18V, TB_18H, TB_23V, TB_23H,
TB_36V, TB_36H, TB_89V, TB_89H,
GR_36V_18V, GR_89V_18V,
PR_18, PR_23, PR_36, PR_89
```

### 2. context

在上面基础上加入：

```text
Year, Month, Day, Hour,
Latitude, Longitude, Distance_Km,
Air_Temp_C, Air_Pressure_mbar
```

### 3. context no pressure

和上面一样，但把：

```text
Air_Pressure_mbar
```

去掉。

## 为什么要单独试一个“去掉气压”的版本

不是说气压一定没用，也不是说以后绝对不能用，而是我怀疑它会带来两个问题：

1. 气压这一列有缺失值，所以一旦把它放进特征，很多样本会被删掉。
2. 如果模型加了气压之后结果变化很大，我需要分清楚，到底是“气压本身提供了有用信息”，还是“样本量变化影响了结果”。

所以我做这个实验的意思其实是：

```text
把 Air_Pressure_mbar 去掉，看看模型表现会不会更稳定，
以及判断之前 context 版本的变化到底有多少是由缺失样本引起的。
```

所以这里不是“必须剔除气压”，而是把它当作一个对照实验。

## 为什么要做 leave-one-buoy-out

这个实验的意思是：

```text
每次拿一个 buoy 全部样本当测试集，
剩下所有 buoy 当训练集。
```

也就是说，如果这次留出的是 `2015A`，那模型训练时完全看不到 `2015A` 的数据，最后只在 `2015A` 上测试。

这个实验比普通随机切分更严格，因为随机切分时，同一个浮标不同时间点的数据可能会同时出现在训练集和测试集里，模型容易“见过很像的样本”。

所以 leave-one-buoy-out 更接近我真正关心的问题：

```text
模型对没见过的浮标到底行不行。
```

## 结果一：tb only

### random split

```text
RMSE = 0.0568 m
MAE  = 0.0191 m
R2   = 0.879
```

### buoy held-out

```text
RMSE = 0.2116 m
MAE  = 0.1756 m
R2   = -4.33
```

### 我的理解

只用亮温和衍生亮温特征时，随机切分结果还不错，但按浮标留出以后效果明显变差。说明模型在“见过类似样本”的情况下能学到关系，但跨浮标泛化不行。

## 结果二：context

### random split

```text
RMSE = 0.0140 m
MAE  = 0.0079 m
R2   = 0.994
```

### buoy held-out

```text
RMSE = 0.1598 m
MAE  = 0.1393 m
R2   = -3.00
```

### 我的理解

加了时间、经纬度、距离和气象信息以后，随机切分结果非常漂亮；按浮标留出也比 tb only 好一点，但还是不理想。

我的理解是：

```text
这些 context 特征确实提供了有用信息，
但它们并没有从根本上解决跨浮标泛化问题。
```

## 结果三：context no pressure

### random split

```text
RMSE = 0.0182 m
MAE  = 0.0066 m
R2   = 0.988
```

### buoy held-out

```text
RMSE = 0.1784 m
MAE  = 0.1677 m
R2   = -2.79
```

### 我的理解

去掉气压以后，随机切分还是很好，按浮标留出比最开始的 tb only 好，但没有比“带气压的 context”明显更强。

所以我现在的判断是：

```text
气压不是当前最核心的矛盾。
真正更大的问题还是不同浮标之间的分布差异。
```

## 结果四：leave-one-buoy-out

这部分我觉得是这次最值得讲的新增内容，因为它把“泛化问题”拆得更细了。

不同 buoy 的误差差异很大。

### 相对容易的 buoy

```text
2015B: RMSE 0.084
2015A: RMSE 0.096
2015I: RMSE 0.105
2015J: RMSE 0.120
```

### 中等

```text
2014E: RMSE 0.135
2014F: RMSE 0.147
2015E: RMSE 0.161
2014D: RMSE 0.161
```

### 最难的 buoy

```text
2014B: RMSE 0.204
2015G: RMSE 0.218
2014C: RMSE 0.273
2015D: RMSE 0.391
```

### 我的理解

这说明模型不是“对所有浮标都差”或者“对所有浮标都好”，而是：

```text
不同 buoy 的难度差异很大。
```

所以现在的问题更像是：

```text
某些浮标的数据分布和其他浮标差得比较远，
导致模型在这些 buoy 上外推能力不足。
```

这个结论比单纯说“R2 不好”更具体，也更像是在往前推进。

## 这次我想怎么讲

我想把故事讲成下面这样：

### 开头

> 上次我先跑了一个随机森林，当时随机切分结果还不错。这次我主要想检查一下，这个结果到底是不是可靠的。

### 中间

> 所以这次我把验证方式做得更严格了一点，不光看 random split，也看按浮标留出的结果。另外我还比较了只用亮温、加入 context、以及去掉气压后的 context 三种输入方案。

### 结果

> 现在比较一致的现象是，random split 都很好，但一旦按浮标留出，性能就明显下降。说明模型在浮标内部拟合得不错，但跨浮标泛化还有问题。

### 新增亮点

> 为了进一步看问题出在哪，我又做了 leave-one-buoy-out。结果发现不同浮标之间差异很大，有些 buoy 预测还可以，有些 buoy 的 RMSE 会明显偏大。这说明当前更值得关注的是不同 buoy 的分布偏移，而不是单纯换一个更复杂模型。

### 收尾

> 所以下一步我更倾向于先继续做跨数据源或者更细的泛化验证，再决定是不是要把物理约束或者 PINN 加进来。

